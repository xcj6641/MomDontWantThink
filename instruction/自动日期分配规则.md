# 自动日期分配规则（最终确认版）

## 1. 目标

自动日期分配用于：

- 首次生成本周辅食计划
- 用户增加备餐数量，并勾选「自动重新分配日期」
- 用户删除某个备餐后，需要重新分配该备餐原有日期
- 用户确认时发现存在未分配日期，需要自动补齐

自动分配的目标优先级为：

1. 保证周一到周日每天都有备餐
2. **full_reset** 时使用固定默认分配方案
3. **fill_empty / redistribute_removed** 时，优先让各备餐天数尽量均衡；天数相同再考虑连续性

## 2. 基本概念

### 2.1 dayAssignments

表示每一天归属哪个备餐，例如：

```json
{
  "mon": 1,
  "tue": 1,
  "wed": 2,
  "thu": 2,
  "fri": 2,
  "sat": 3,
  "sun": 3
}
```

### 2.2 menuCount

当前备餐数量，范围 1～7。

### 2.3 分配模式

- full_reset
- fill_empty
- redistribute_removed

## 3. full_reset 规则（固定分配方案）

### 3.1 使用场景

- 首次生成本周计划
- 用户增加备餐数量，并勾选「自动重新分配日期」

### 3.2 固定方案表

| 套数 | 分配 |
|------|------|
| 1 套 | 7 |
| 2 套 | 4-3 |
| 3 套 | 2-3-2 |
| 4 套 | 2-2-2-1 |
| 5 套 | 2-1-2-1-1 |
| 6 套 | 1-1-2-1-1-1 |
| 7 套 | 1-1-1-1-1-1-1 |

### 3.3 映射结果

- **menuCount = 1**：mon~sun 均为 1
- **menuCount = 2**：mon~thu=1，fri~sun=2
- **menuCount = 3**：mon,tue=1；wed,thu,fri=2；sat,sun=3
- **menuCount = 4**：mon,tue=1；wed,thu=2；fri,sat=3；sun=4
- **menuCount = 5**：mon,tue=1；wed=2；thu,fri=3；sat=4；sun=5
- **menuCount = 6**：mon=1；tue=2；wed=3；thu=4；fri=5；sat,sun=6
- **menuCount = 7**：mon~sun 依次 1~7

### 3.4 full_reset 行为

直接返回当前 menuCount 对应的固定方案（建议实现 `getDefaultDayAssignments(menuCount)`）。

## 4. fill_empty 规则

### 4.1 使用场景

用户手动调整日期后某些日期为空，点击确认时触发自动补齐。

### 4.2 分配优先级

当某一天为空白时，按以下顺序决定归属：

1. **Step 1**：先看当前各备餐绑定天数 → 选择 **当前绑定天数最少的备餐**（得到候选集合 candidateMenus）
2. **Step 2**：若有多个备餐天数相同，再考虑连续性：
   - 若前一天已属于某个候选备餐，优先分给它
   - 若后一天已属于某个候选备餐，也可优先考虑
   - 前后都存在但不同时，优先前一天的备餐
   - 仍无法判断则选编号更小的备餐

### 4.3 示例

当前：mon,tue=1；wed=空；thu,fri=2；sat,sun=3。备餐1/2/3 均为 2 天。处理 wed：三天数相同，前一天 tue=1 → wed 分给 1。结果：mon,tue,wed=1；thu,fri=2；sat,sun=3。

## 5. redistribute_removed 规则

### 5.1 使用场景

用户删除某个备餐，该备餐原有日期需要重新分配。

### 5.2 行为

- 被删除备餐绑定的日期设为 null
- 后续备餐重新编号
- 对这些 null 日期执行 **fill_empty**

### 5.3 示例

原：mon,tue=1；wed,thu=2；fri,sat=3；sun=4。删除备餐4 后 sun=null，备餐1/2/3 各 2 天。处理 sun：天数相同，前一天 sat=3 → sun 分给 3。最终 sun=3。

## 6. 重要约束

- **必须**：周一到周日每一天都必须属于一个备餐
- **不必须**：每个备餐都至少分配一天（允许「暂未使用」）

## 7. 增加备餐时

- **勾选「自动重新分配日期」**：使用 full_reset
- **未勾选**：保持原有日期分配，新增备餐只生成内容、不自动绑定日期，显示为「暂未使用」

## 8. 技术实现建议

- 固定方案：`getDefaultDayAssignments(menuCount)` 返回 7 天归属数组
- 主分配：`autoAssignDays({ dayAssignments, menuCount, mode, removedMenuIndex })`

## 9. 伪代码（最终版）

```text
function getDefaultDayAssignments(menuCount):
    switch menuCount 1..7 返回对应固定数组（周一～周日）

function autoAssignDays(dayAssignments, menuCount, mode, removedMenuIndex = null):
    if mode == "full_reset": return getDefaultDayAssignments(menuCount)
    if mode == "redistribute_removed":
        被删备餐的日期置 null；编号 > removedMenuIndex 的减 1
        mode = "fill_empty"
    if mode == "fill_empty":
        counts = 各备餐已分配天数
        for each day in days:
            if dayAssignments[day] is null:
                minCount = 最少天数
                candidateMenus = 所有 count == minCount 的备餐
                if candidateMenus 含 prevMenu: target = prevMenu
                else if candidateMenus 含 nextMenu: target = nextMenu
                else: target = candidateMenus 中编号最小
                dayAssignments[day] = target; counts[target] += 1
        return dayAssignments
```

## 10. 一句精简描述

full_reset 永远使用固定默认分配方案；fill_empty 和 redistribute_removed 时，优先把空白日期分配给当前绑定天数最少的备餐，天数相同再考虑前后相邻日期的连续性，最后按编号最小。系统只要求每天有备餐，不要求每个备餐都被使用。
