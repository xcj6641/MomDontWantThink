# 初始信息页设计风格规范（Onboarding）

本文档从当前实现中提取 **首页初始界面**（`pages/home`）、**今日辅食调试初始页**（`pages/todayFood` 内 `.onboarding`）以及 **与欢迎卡同源的头部信息栏**（如今日辅食常规页 `.baby-bar`）共用的视觉与布局规则，供后续页面复刻或评审对照。

**实现参考**

- `miniprogram/pages/home/home.wxss`（`.container-onboarding`、`.onboarding`、`.welcome-card` 等）
- `miniprogram/pages/todayFood/todayFood.wxss`（`.onboarding` 前缀块与主内容区 `.baby-bar`，与 home 欢迎区对齐）
- 结构参考：`miniprogram/pages/home/home.wxml` 中 `showInitial` 区块；`miniprogram/pages/todayFood/todayFood.wxml` 中宝宝信息条

## 一、整体定位

| 维度 | 说明 |
|------|------|
| 气质 | 温和、清晰、偏「母婴记录」向；用浅绿建立信任，白卡片承载表单 |
| 层级 | 页面灰底 → 顶部浅绿欢迎区 → 白底表单卡 → 底栏白底固定主按钮 |
| 装饰 | 欢迎区保留 **2rpx 淡绿描边**；表单卡 **2rpx 浅灰描边 + 轻阴影**，避免厚重投影 |
❗母婴感 = 轻柔颜色 + 圆润形态 + 情绪表达 + 留白

## 七、组件级说明

### 7.1 欢迎卡（`.welcome-card`）

- 横向 flex：`align-items: flex-start`，左图标 + 右文案区（`min-width: 0` 防挤压换行异常）。
- 图标与正文列间距：**`gap: 20rpx`**（与 **§7.2** Grid 的 **`column-gap: 20rpx`** 一致）。
- 背景 `welcome-bg`，描边 `welcome-border`，圆角 32rpx，内边距 **32rpx**。

### 7.2 同源头部信息栏（多行变体，例：今日辅食 `.baby-bar`）

当需要在**同一视觉块**内展示多行（如第一行「宝宝 · 月龄」、第二行「过敏：…」），且与欢迎卡 **图标大小、图标-正文间距、主/副标题字号字色** 完全一致时，采用下列约定（已实现于 `todayFood` 宝宝信息条）：

1. **外观**  
   - 与 `.welcome-card` 相同：**背景 / 描边 / 圆角 / 内边距 32rpx**（`welcome-bg`、`welcome-border`、32rpx）。

2. **布局（推荐 CSS Grid）**  
   - `display: grid`  
   - `grid-template-columns: max-content 1fr`（首列宽度由 emoji 实际占位决定，**等价于欢迎卡不在图标上写死宽度**）  
   - `column-gap: 20rpx`（**与 `.welcome-card` 的 `gap: 20rpx` 一致**）  
   - `row-gap: 8rpx`（与欢迎区内「标题与副标题」纵向 **8rpx** 节奏一致）  
   - `align-items: start`

3. **图标**  
   - 与 `.welcome-card-icon`：**`font-size: 44rpx`，`line-height: 1`**。  
   - **不要**给图标列设置大于字形的固定 `width`（避免横向空隙大于 20rpx）。

4. **第二行与「宝宝」左对齐**  
   - 第二行第一列使用与首行**相同字符**的占位（如透明 **`opacity: 0`** 的同一 emoji），**字号、行高与首行图标一致**，使第二列文案与第一行「主标题」左缘对齐。  
   - 若仅有第二行文案（无宝宝行），第二行应 **`grid-column: 1 / -1`** 通栏展示。

5. **文字层级（与欢迎卡逐条对应）**  
   - 主行：等同 **`.welcome-card-title`** — `34rpx`、`font-weight: 600`、`#2F5E3B`、`display: block`（不设额外 `line-height` 覆盖时与首页一致）。  
   - 次行：等同 **`.welcome-card-subtitle`** — `26rpx`、`#7A8A7E`、`line-height: 1.45`、`display: block`。

6. **字体**  
   - 整块容器使用与 **`page` 相同 `font-family`**（见上文「字体栈」），与初始信息页头部一致。

### 7.3 表单卡片（`.form-card`）

- 白底、浅灰描边、轻阴影；圆角 32rpx。
- 头部：标题 + 「必填」或「可选」徽标同一行 flex。

### 7.4 日期选择器触发区（`.picker-trigger`）

- 整行可点；灰底 + 细描边；按压 `opacity: 0.9`。

### 7.5 过敏标签（`.allergen-chip`）

- 浅绿底 + 品牌绿字；右侧删除「×」略透明（0.75），按下不透明。

### 7.6 添加过敏（`.btn-add-allergen`）

- 与 chip 同色系实心块，非描边幽灵按钮；小程序 `button` 需去默认边框（`::after { border: none }`）。

### 7.7 底栏主按钮（`.btn-primary-cta`）

- 全宽，**高度 96rpx**；**135° 线性渐变** `#7FB77E → #6BA86A`；白字、圆角 24rpx。
- 按下 **opacity 0.88**；禁用灰底 `#D0D4D8`、字 `#9CA3A8`。

## 八、与全局《风格规范》的差异提示

全局规范倾向：**12px 卡片圆角、无描边、主按钮纯色 36px 高**。初始信息页保留 **更大圆角（32rpx/24rpx）**、**细描边**、**更高主按钮（96rpx）** 与 **绿渐变 CTA**，属于产品定义的 **Onboarding 子品牌带**；新页面若要求「与初始页一致」，以 **本文档** 为准；若要求「全应用统一 DS」，则需在迁移时逐项替换并回归验收。
